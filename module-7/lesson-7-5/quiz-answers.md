# Quiz Answers: Performance Testing

## Question 1
**What is LCP?**

**Correct Answer**: B) Largest Contentful Paint

**Explanation**: LCP measures when the largest visual element becomes visible to the user.

---

## Question 2
**What is target LCP?**

**Correct Answer**: B) < 2.5s

**Explanation**: Google recommends LCP of 2.5 seconds or less for good user experience.

---

## Question 3
**What is code splitting?**

**Correct Answer**: B) Dividing code into chunks loaded when needed

**Explanation**: Code splitting reduces initial bundle size by loading features only when accessed.

---

## Question 4
**What is tree shaking?**

**Correct Answer**: A) Removing unused code

**Explanation**: Tree shaking is a build optimization that removes unused code from the final bundle.

---

## Question 5
**What does Lighthouse measure?**

**Correct Answer**: C) Multiple aspects: performance, accessibility, SEO

**Explanation**: Lighthouse audits performance, accessibility, best practices, SEO, and PWA capabilities.

---

## Question 6
**What's CLS?**

**Correct Answer**: B) Cumulative Layout Shift

**Explanation**: CLS measures unexpected layout changes during page load.

---

## Question 7
**What should CLS score be?**

**Correct Answer**: A) < 0.1

**Explanation**: Google targets CLS of less than 0.1 for good user experience.

---

## Question 8
**What is lazy loading?**

**Correct Answer**: B) Deferring resource load until needed

**Explanation**: Lazy loading improves initial load time by loading resources only when needed (e.g., images, routes).

---

## Question 9
**What is FID?**

**Correct Answer**: C) First Input Delay

**Explanation**: FID measures how quickly browser responds to user interactions.

---

## Question 10
**What tool audits web performance?**

**Correct Answer**: C) Lighthouse

**Explanation**: Lighthouse is Google's official tool for measuring web performance and quality metrics.
