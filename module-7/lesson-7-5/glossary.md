# Glossary - Performance Testing

## Performance Testing

**Definition**: Testing application speed, efficiency, and resource usage.

---

## Lighthouse

**Definition**: Google's tool for auditing web page performance and quality.

---

## Core Web Vitals

**Definition**: Three metrics Google uses to measure user experience.

---

## LCP (Largest Contentful Paint)

**Definition**: Time when the largest visual element appears on screen.

---

## FID (First Input Delay)

**Definition**: Time between user action and browser response.

---

## CLS (Cumulative Layout Shift)

**Definition**: Measure of unexpected layout changes during page load.

---

## Bundle Size

**Definition**: Total file size of compiled JavaScript code.

---

## Code Splitting

**Definition**: Dividing code into smaller chunks loaded only when needed.

---

## Lazy Loading

**Definition**: Deferring loading of resources until they're needed.

---

## Tree Shaking

**Definition**: Removing unused code during build process.

---

## Performance Audit

**Definition**: Systematic measurement of application performance metrics.

---

## Bottleneck

**Definition**: Part of code/app that limits performance.

---

## Rendering Performance

**Definition**: Speed at which browser renders/updates UI.
