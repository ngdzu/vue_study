# Glossary - Snapshot & Visual Testing

## Snapshot Test

**Definition**: Test that captures component output and fails if output changes.

---

## Snapshot File

**Definition**: File storing saved component output for comparison.

---

## Visual Regression

**Definition**: Unintended visual changes detected by comparing screenshots.

---

## Screenshot Testing

**Definition**: Capturing and comparing visual appearance of components.

---

## Snapshot Update

**Definition**: Intentionally updating saved snapshots when changes are wanted.

---

## Regression Detection

**Definition**: Automatically detecting unintended changes from previous versions.

---

## Percy / Chromatic

**Definition**: Visual testing tools that detect UI changes automatically.

---

## Golden Master

**Definition**: Reference (snapshot) used to detect changes.

---

## False Positive

**Definition**: Test failure on acceptable/intentional changes.

---

## Approval Testing

**Definition**: Reviewing and approving output changes explicitly.
