# Glossary - E2E Testing with Cypress

## E2E (End-to-End) Test

**Definition**: Test that verifies complete user workflows through real browser interaction.

---

## Cypress

**Definition**: An E2E testing framework that runs in real browsers and provides excellent debugging tools.

---

## Base URL

**Definition**: The starting URL for Cypress tests, configured in cypress.config.ts.

---

## Selector

**Definition**: A way to find elements on the page (CSS selectors, text content, test IDs).

---

## Data Test ID

**Definition**: A custom HTML attribute for selecting elements in tests reliably.

---

## Cy.visit()

**Definition**: Cypress command to navigate to a URL.

---

## Cy.get()

**Definition**: Cypress command to select elements using CSS selectors.

---

## Cy.contains()

**Definition**: Cypress command to find elements by text content.

---

## Cy.type()

**Definition**: Cypress command to type text into an input field.

---

## Cy.click()

**Definition**: Cypress command to click an element.

---

## Cy.submit()

**Definition**: Cypress command to submit a form.

---

## Assertion

**Definition**: A check that verifies the application state meets expectations.

---

## Should()

**Definition**: Cypress command to assert conditions about elements.

---

## Timeout

**Definition**: How long Cypress waits for element/condition before failing test.

---

## Flaky Test

**Definition**: A test that fails intermittently due to timing issues.

---

## Workflow Test

**Definition**: An E2E test that tests a complete user journey.

---

## Page Object

**Definition**: A pattern for organizing selectors and interactions for a page.
